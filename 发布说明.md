# 灵境网站发布说明

## 你只需要记住三件事

1. 线上网站只认 `index.html`。
2. 草稿、备份、试验文件不要直接发布。
3. 发布时不要用 `git add .`，只上传你确认要发布的文件。

## 文件怎么放

- `index.html`：正式首页，线上网站正在用它。
- `drafts/`：草稿区，Claude 或 VS Code 可以随便改这里。
- `backups/`：备份区，每次大改前可以把旧文件复制进去。
- `assets/`、`models/`：网站正式资源区，不确定时不要乱删。

## 推荐工作流程

先在草稿里改，例如：

```bash
cp index.html drafts/index-test.html
```

用 VS Code 或 Claude 修改 `drafts/index-test.html`。

确认满意后，再覆盖正式首页：

```bash
cp drafts/index-test.html index.html
```

发布前先备份：

```bash
cp index.html "backups/index-$(date +%Y%m%d-%H%M).html"
```

然后发布：

```bash
cd "/Users/zzziccgeo/Desktop/Lingjing- interact"
export PATH="/Applications/GitHub Desktop.app/Contents/Resources/app/git/libexec/git-core:$PATH"
git status
git add index.html
git commit -m "update homepage"
git push origin main
```

如果还新增了确定要上线的图片，比如 `assets/new-image.jpg`：

```bash
git add index.html assets/new-image.jpg
```

## 遇到不确定的情况

发布前把 `git status` 的结果发给 Codex，说：

```text
帮我检查这次应该上传哪些文件
```

不要自己随便运行 `git add .`。
